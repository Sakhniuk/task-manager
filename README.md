# Лабораторна робота №1 — CRUD API

## Мета роботи

Реалізувати базовий REST API без використання складних архітектурних підходів.
Проект є початковою версією для подальшого вдосконалення.

---

## Домен

Система керування задачами (Task Manager)

---

## Автори

* Євтєєва Вероніка — юзкейси, тести, документація
* Сахнюк Юля — backend, API, авторизація

---

## Технології

* Node.js
* Express.js
* JSON Web Token (JWT)
* Jest
* Supertest

```

  "email": "test@test.com",
  "password": "123456"
}
```

У відповідь повертається токен доступу.

---

## REST API (Tasks)

### POST /tasks

Створення задачі

```
{
  "title": "Test task",
  "deadline": "2030-01-01"
}
```

### GET /tasks

Отримати список задач

### GET /tasks/:id

Отримати одну задачу

### PUT /tasks/:id

Оновити задачу

### DELETE /tasks/:id

Видалити задачу

---

## Захист API

Ендпоінти `/tasks` доступні тільки для авторизованих користувачів.

Заголовок запиту:

```
Authorization: Bearer <token>
```

---

## Обробка помилок

* 400 — невалідні дані
* 401 — не авторизований
* 404 — ресурс не знайдено
* 409 — конфлікт

---

## Інваріанти

* title не може бути пустим
* deadline не може бути в минулому
* задача повинна існувати для оновлення та видалення
